# Serenity BDD 5.3.9 Release Notes

**Page Source Recording, Security Fixes, and Screenplay Improvements**

This release re-enables page source recording with per-screenshot links, fixes two security vulnerabilities, and resolves a Screenplay reporting issue.

## Bug Fixes

### Page Source Recording Disabled (#3719)

The `serenity.store.html` configuration property (`ALWAYS`, `FAILURES`, `NEVER`) had no effect because `PageSourceRecorder.shouldRecordPageSource()` was hardcoded to `return false`. Removed the hardcoded gate and fixed the default in `SoundEngineer` to `false` so recording is properly controlled by configuration (default: `FAILURES`).

Additionally, each screenshot slide in the carousel now includes an "HTML Source" button linking to the corresponding page source file when available.

### Raw HTML Leaking in Screenshot Carousel

Screenshot descriptions containing quote characters could break the `<img title="...">` attribute on the last carousel slide, rendering raw HTML as visible text. Fixed by applying Freemarker `?html` escaping to captions.

### QuestionWithDefinedSubject Missing toString() (#3661)

Questions created via `QuestionBuilder` displayed as `QuestionWithDefinedSubject@hashcode` in reports instead of the defined subject. Added `toString()` that returns the subject.

## Security Fixes

### jackson-core (GHSA-72hv-8253-57qq)

Updated jackson-core from 2.20.1 to 2.21.1 to fix a DoS vulnerability via number length constraint bypass in the async parser.

### plexus-utils (CVE-2025-67030) — Not Updated

The plexus-utils vulnerability (directory traversal in `extractFile`) cannot be fixed at this time. The 4.x line removes the `org.codehaus.plexus.util` package entirely, which is a breaking change for the Maven plugin. The vulnerable code path is not exercised by the Serenity Maven plugin.

## Affected Components

| Component | Change |
|-----------|--------|
| `serenity-core` | Re-enable page source recording; fix `SoundEngineer` default |
| `serenity-model` | Add `htmlSourceName` to `Screenshot` for per-slide source links |
| `serenity-report-resources` | Escape carousel captions; add per-slide HTML Source button |
| `serenity-screenplay` | Add `toString()` to `QuestionWithDefinedSubject` |
| `pom.xml` | jackson-core 2.20.1 → 2.21.1 |

## Upgrade

Using the BOM (recommended):

```xml
<dependencyManagement>
    <dependencies>
        <dependency>
            <groupId>net.serenity-bdd</groupId>
            <artifactId>serenity-bom</artifactId>
            <version>5.3.9</version>
            <type>pom</type>
            <scope>import</scope>
        </dependency>
    </dependencies>
</dependencyManagement>
```

Or update the version directly:

```xml
<dependency>
    <groupId>net.serenity-bdd</groupId>
    <artifactId>serenity-core</artifactId>
    <version>5.3.9</version>
</dependency>
```

No code changes are required. This is a drop-in replacement for 5.3.8.
