# Serenity BDD 5.3.8 Release Notes

**Bug Fixes for Screenshot Navigation, Page Object Injection, and Screenplay Improvements**

This release fixes several bugs including broken screenshot carousel navigation, a `@FindBy` NullPointerException regression, and raw HTML leaking in report screenshots. It also includes improvements to Screenplay Interaction handling and Cucumber parallel mode support.

## Bug Fixes

### Screenshot Navigation Opens Wrong Screenshot (#3722)

Clicking a screenshot thumbnail in the step table could open the wrong screenshot in the carousel, with the offset growing worse the more group steps were in the test. The `screenshotCount` index in the report template was being double-incremented for group steps — once when rendering child steps and again after the group's nested table. Removed the redundant increment.

### Raw HTML Leaking in Screenshot Carousel

The last screenshot slide could display raw HTML like `(completed): SUCCESS" width="0" />` when a step description contained quote characters. These broke out of the `<img title="...">` attribute, rendering the rest of the tag as visible text. Fixed by applying HTML escaping to screenshot captions.

### @FindBy NullPointerException (#3714)

A regression introduced in v4.2.19 (PR #3575) accidentally inverted the WebDriver injection logic in `PageObjectDependencyInjector.updatePageObject()`. The condition was changed from `if (hasAnInstantiatedDriver())` to `if (!hasAnInstantiatedDriver())`, meaning the driver was only set on page objects when no driver was available. This left `@FindBy` annotated fields uninitialized, causing NullPointerExceptions. Reverted to the correct logic.

### Actor Abilities/Facts Not Recorded in Cucumber Parallel Mode (#3731)

Actor abilities and facts were not being recorded when running Cucumber scenarios in parallel. Added `AssignAbilityToActorEvent` and `AssignFactToActorEvent` to ensure these are dispatched through the `StepEventBus` correctly in parallel execution. (Contributed by @lsmarsden)

## Improvements

### Screenplay Interaction Alignment (#3726)

`AnonymousInteraction` methods now return instances that implement the `Interaction` interface, aligning the behavior with `Task`. This ensures consistent type handling when working with anonymous interactions in Screenplay tests. (Contributed by @TrimbleGlob)

## Deprecations

### Jira Integration Modules

The `serenity-jira-plugin` and `serenity-jira-requirements-provider` modules are now deprecated. Atlassian's API token restrictions (mandatory expiration, limited scopes, and permission changes) make it impractical to reliably test and maintain these integrations. The modules remain in the codebase for existing users but are no longer actively maintained. If Jira integration is important to your project, please contact us about a [commercial support package](https://www.serenitydojo.academy/serenity-bdd-support-packages).

## Affected Components

| Component | Change |
|-----------|--------|
| `serenity-report-resources` | Fix screenshot index drift for group steps; escape captions in carousel |
| `serenity-core` | Fix inverted driver injection for `@FindBy` page objects |
| `serenity-screenplay` | Align `AnonymousInteraction` return types with `Interaction` interface |
| `serenity-model` | Add `AssignAbilityToActorEvent` / `AssignFactToActorEvent` for parallel Cucumber |
| `serenity-jira-plugin` | Deprecated; tests disabled; improved error messages with query URLs |
| `serenity-jira-requirements-provider` | Deprecated; tests disabled |

## Upgrade

Using the BOM (recommended):

```xml
<dependencyManagement>
    <dependencies>
        <dependency>
            <groupId>net.serenity-bdd</groupId>
            <artifactId>serenity-bom</artifactId>
            <version>5.3.8</version>
            <type>pom</type>
            <scope>import</scope>
        </dependency>
    </dependencies>
</dependencyManagement>
```

Or update the version directly:

```xml
<dependency>
    <groupId>net.serenity-bdd</groupId>
    <artifactId>serenity-core</artifactId>
    <version>5.3.8</version>
</dependency>
```

No code changes are required. This is a drop-in replacement for 5.3.7.
