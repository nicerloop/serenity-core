# Serenity BDD 5.3.11 Release Notes

**Report Encoding Fix and Soft Assertion Improvements**

This release fixes character encoding in report Examples tables and adds soft assertion support for question-predicate assertions.

## Bug Fixes

### Character Encoding in Examples Tables (#3673)

Special characters (accents, tildes, etc.) were displaying incorrectly in the Examples table section of Serenity reports, while rendering correctly in other report sections. The root cause was double HTML escaping: `Formatter.plainHtmlCompatible()` converted accented characters to HTML entities (e.g., `á` to `&aacute;`), and then Freemarker's `<#outputformat 'HTML'>` directive escaped them again, producing `&amp;aacute;`. Fixed by adding `?no_esc` to the Freemarker template to prevent re-escaping of already-escaped content.

### `Ensure.that(question, predicate)` Ignores Soft Assertions (#3710)

The `Ensure.that(question, predicate)` overload threw assertion errors immediately even when soft assertions were enabled via `Ensure.enableSoftAssertions()`. This happened because the method used `AnonymousPerformableFunction`, which has no soft assertion awareness. Replaced with a new `PerformableQuestionPredicate` class that routes failures through the same `handleException()` mechanism used by all other Ensure assertion classes.

**Before (broken):**
```java
Ensure.enableSoftAssertions();
actor.attemptsTo(
    Ensure.that(question1, value -> value.startsWith("X")), // threw immediately
    Ensure.that(question2, value -> value.endsWith("Y"))     // never reached
);
Ensure.reportSoftAssertions();
```

**After (fixed):**
Both assertions are now evaluated and all failures are reported together when `reportSoftAssertions()` is called, consistent with other Ensure methods.

## Affected Components

| Component | Change |
|-----------|--------|
| `serenity-report-resources` | Fix double HTML escaping in Examples table templates (default.ftl, junit5/default.ftl) |
| `serenity-ensure` | New `PerformableQuestionPredicate` for soft-assertion-aware question-predicate checks |

## Upgrade

Using the BOM (recommended):

```xml
<dependencyManagement>
    <dependencies>
        <dependency>
            <groupId>net.serenity-bdd</groupId>
            <artifactId>serenity-bom</artifactId>
            <version>5.3.11</version>
            <type>pom</type>
            <scope>import</scope>
        </dependency>
    </dependencies>
</dependencyManagement>
```

Or update the version directly:

```xml
<dependency>
    <groupId>net.serenity-bdd</groupId>
    <artifactId>serenity-core</artifactId>
    <version>5.3.11</version>
</dependency>
```

No code changes are required. This is a drop-in replacement for 5.3.10.
